# 🐾 Discipline Dimo — 自律的迪莫

**一个会陪你养成好习惯的桌面小宠物。**

迪莫会像真正的伙伴一样，在桌面上呼吸、眨眼、对你做出不同反应。  
双击它就能打开你的专属习惯打卡网页，记录每日坚持，积攒积分，兑换愿望。



---

## ✨ 功能详解

### 🎈 迪莫悬浮球交互规则

| 你的操作 | 迪莫的反应 | 说明 |
|----------|------------|------|
| **单击** 头像 | 眨眼（wink） | 仅表情，不打开主窗口 |
| **双击** 头像 | 开心表情 + 打开主界面 | 打开打卡网页 |
| **按住并拖动** 头像 | 惊讶表情 | 只要按住超过 0.2 秒并移动，就会触发 |
| **闲置 10 秒** 不操作 | 打哈欠（犯困） | 任何操作（单击/双击/拖动）都会唤醒 |
| 呼吸、自动眨眼 | 一直持续 | 无需操作，永远在呼吸，每 4 秒自动眨眼 |


此外，迪莫还会**自动眨眼**（每 4 秒一次）和**持续呼吸**（永远轻微缩放），非常生动。

### 📅 打卡网页功能
- **每日打卡**：点击“今日打卡”按钮，增加连续天数和积分。
- **多旗帜系统**：可以同时立多个 Flag（如“运动：每天跑步 3 公里”）。每个 Flag 有专属鼓励语，完成后会点亮地图上的里程碑。
- **愿望与兑换**：用积分兑换“一杯奶茶”、“一顿饭”或“实现一个愿望”。愿望需要先提交，再由你手动兑换。
- **积分奖励**：每次打卡得 1 积分，连续 7 天额外奖励 1 积分。
- **日历视图**：查看哪些日子打了卡，打卡日会用铅笔画风圈起来。
- **里程碑地图**：每完成一个 Flag，地图上的一个里程碑就会被点亮（从迷雾中显现）。
- **PushPlus 提醒**：在“我的”页面配置你的 [PushPlus Token](https://www.pushplus.plus/)，之后每晚 22:00 如果你还没打卡，就会收到微信提醒。

### 🔐 用户系统
- 注册 / 登录（用户名 + 密码，纯文本暂存，请勿使用重要密码）
- 修改密码
- 上传头像（支持 jpg/png/gif，最大 2MB）
- 所有用户数据独立（打卡记录、积分、Flag、愿望等）

## 🚀 如何安装和运行

### 前提条件
- **Node.js**：版本 16 或以上。[点击下载](https://nodejs.org/)
- **npm**：随 Node.js 自动安装。

### 步骤 1：获取代码
打开终端，执行：
```bash
git clone https://github.com/你的用户名/Discipline-Dimo.git
cd Discipline-Dimo
```
步骤 2：安装依赖
```bash
npm install
```
这会自动安装所有需要的包（Express, Electron, SQLite, 等）。

步骤 3：启动应用
```bash
npm start
```
你会看到终端输出：
```bash
✅ 服务已启动，端口：8080
[后端] ...
```
同时，桌面会出现迪莫悬浮球。
步骤 4：开始使用
双击迪莫 → 打开打卡网页（浏览器窗口）。
注册账号（随便填，数据存在你电脑本地）。
开始打卡、立 Flag、兑换愿望。
步骤 5：关闭应用
在终端按 Ctrl + C，后端和悬浮球都会退出。
---
###🧠 如何保证应用一直正常运行？
⚠️ 重要：必须保持终端打开
npm start 启动的是一个命令行进程，一旦关闭终端窗口，应用就会退出。
解决办法：
最小化终端窗口，不要关闭它。
或者使用 后台运行 的方式（但不推荐，因为不方便关闭）。
如果想“双击图标”就能运行（不需要终端）
你需要将应用打包成独立的 .app（macOS）或 .exe（Windows）。打包命令：
```bash
npm run dist
```
然后会在 dist 文件夹生成 DMG 或 EXE 安装包。
注意：打包后可能出现图片加载问题，请参考常见问题第 7 条。

端口冲突

如果 8080 端口被其他程序占用，可以修改 server.js 中的 PORT 变量（例如改成 3000），然后重新启动。

数据库文件

首次运行会自动创建 data.db 文件（在项目根目录）。
不要删除它，否则所有用户数据会丢失。
如果你想把数据迁移到另一台电脑，只需复制这个文件。

###❓ 常见问题与解答

1. 运行 npm start 提示 concurrently: command not found

原因：依赖未安装完整。
解决：执行 npm install，等待安装完成后再试。

2. 迪莫悬浮球显示“迪莫”文字，而不是图片

原因：图片文件缺失或路径错误。
解决：确保 public/images/ 目录下有 dimo.png, dimo-happy.png, dimo-surprised.png, dimo-sleepy.png, dimo-wink.png。如果没有，请从 electron/ 文件夹复制一份过去。

3. 双击迪莫无法打开网页

原因：后端服务未启动或端口不对。
解决：检查终端是否显示 ✅ 服务已启动，端口：8080。如果没有，可能是端口被占用，杀死占用进程或修改端口。

4. 打卡后积分没有增加

原因：数据库写入失败。
解决：查看终端是否有红色错误信息。检查 data.db 文件是否存在且有写入权限。

5. 如何修改密码或头像？

在“我的”页面，有修改密码和上传头像的表单。头像会保存在 public/uploads/avatars/ 目录。

6. 我想把数据迁移到另一台电脑

复制整个项目文件夹（包括 data.db）到新电脑，然后在新电脑上执行 npm install 和 npm start 即可。

7. 打包后迪莫图片不显示

原因：打包后静态文件路径变化。
建议：继续使用 npm start 方式运行，避免打包。如果你必须打包，请参考 Electron 打包静态资源指南。
